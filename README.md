# xmln - home

项目使用 Vue3 + Vite 构建

克隆下项目后需要在项目的根目录下执行命令安装所需依赖

安装依赖：`npm install`

运行项目：`npm run dev`

项目构建：`npm run build`

## 配置信息

在根目录下有 .env 和 homeConfig.js 这两个文件，这是这个项目的配置文件

.env：这个里面存放这站点的基本信息

homeConfig.js：这个里面存放这所有内容信息，采用对象的形式存储

### homeConfig.js

#### 开关配置

在这个文件中你会看到很多的 enable 或者以它结尾的配置，他是一个开关，用于让页面决定是否展示某个模块的内容，如：

```javascript
// 备案信息
ICP: {
    enable: true,
    info: 'xxxxxx'
}
```

如上所示的配置，当 enable 为 true 时，页面会展示这个配置的内容，当 enable 为 false 时，页面不会展示这个配置的内容

#### 文本内容配置

在配置文件中，在指定的配置项内，可以使用一些简单的语法格式，项目会自动编译成 HTML 代码，目前所支持的格式有

| 格式        | 标签                       |
|-----------|--------------------------|
| "h1:xxx"  | 一级标题                     |
| "h1:xxx"  | 二级标题                     |
| "h1:xxx"  | 三级标题                     |
| "p:xxx"   | 正常文本                     |
| "img:xxx" | 图片内容，xxx 为图片链接（也可以是相对路径） |

#### Icon 配置（图标配置）

在一些配置项中需要使用到图标比如：社交部分、站内导航部分......

本项目整合了阿里巴巴的图标库，使用步骤：

1. 在[iconfont](https://www.iconfont.cn)上搜索你需要的图标，然后添加到项目中（没有就创建一个）
2. 选择完成之后进入到 iconfont 的项目中，在三个选项中选择 Symbol 这个选项下面会出现一个链接
3. 进入这个链接内，复制里面所有的内容并替换掉本项目的 `/public/js/fonticon.js` 这个文件内原本的内容
4. 最后在需要使用图标的地方的 icon 选项内使用自己的即可

```javascript
{
    name: 'bilibili',
    url: 'https://space.bilibili.com/1304924540',
    icon: 'icon-bilibili'
}
```

#### 图片配置

对于被项目内所有需要使用图片的地方既可以使用超链接也可以使用相对路径，如果使用相对路径的话你需要把所有的图片都放到 `/public/img` 文件夹内，然后在需要图片的地方就可以填写右边的路径 `/img/图片名称.后缀` ，当然你也可以在里面创建一些文件夹用于分类，使用时只需要在 `/img/` 后面加入文件的具体路径即可
